+CSS, Cascading Style Sheets

  -HTML, XML, XHTML 등의 마크업 언어(Markup Language)가 화면에
   표현되는 방법(스타일, 모양, style)을 기술하는 언어
   
    -> HTML에 작성된 태그가 화면에 보여지는 모양을 지정하는 언어
    
  - 부모태그에 스타일을 적용하면 자식태그도 해당 스타일이 적용된다  
  
+ CSS를 HTML에 적용하는 방법 3가지

  1. 인라인(inline) 방식
      - HTML태그의 style속성을 이용하여 스타일을 적용하는 방법
      
      ex)   style="color:red;"
      
      - 장점 : 태그를 보면서 스타일 지정 가능하다
      
      - 단점 : 각각의 태그에 스타일을 일일히 적용해야하기 때문에
              관리가 힘들어진다
      
  2. 임베디드(embedded) 방식
      
      - <head> 태그 내에 <style>태그를 생성하고 그 안에 작성한다
      - CSS 선택자(Selector)를 이용한다
    
      - 장점 : HTML문서에 적용된 모든 스타일을 한번에 확인할 수 있다.
      
      - 단점 : 태그와 스타일이 서로 떨어져 있어서 각각 태그나 스타일만 봐서는 어떤 스타일이 적용될지 예측이 어렵다.
        
      
  3. 별도의 css파일을 로드하는 방식
      
      - @import 코드를 이용하는 방식과 <link>태그를 이용하는 방식이 있다
      
      - 스타일시트를 별도의 외부파일로 생성하여 관리한다
      - HTML파일에서 외부 CSS 파일을 로드해서 사용한다.
      
      - CSS파일의 확장자는 주로 .CSS를 사용한다
      
      - 장점 : 스타일 목적에 맞춰서 파일을 분리할 수 있다
      ex)       main.css, common.css, board.css, login.css, ...
      
      - 단점 : 관리해야하는 파일이 많아진다.
      
      
      ** @import보다 <link>방식을 사용하는 것이 좋다
      ** <link> 하나 당 css파일 하나를 로드하도록 작성한다
